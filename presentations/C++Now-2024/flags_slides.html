--- header: 'Using bits for Boolean options' ## Why do we use the bits of integer types for Boolean options? * Easy to define, e.g. as macros, integer constants or enumerators * Compact representation in memory * Simple & Efficient bulk transfer in interfaces - `unsigned int` can hold up to 16 / 32 different Boolean options * Enforces the usage of names / prevent bool parameters in interfaces > "Boolean arguments loudly declare that the function does more than one thing. They are confusing and should be eliminated." > C. Martin (Uncle Bob), Clean Code * Language support for setting and querying through bitwise operators
